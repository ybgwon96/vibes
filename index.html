포트폴리오와 영업력이 실력을 대체하는 시장. 클라이언트는 진짜 실력자를 찾기 어렵고, 실력 있는 개발자는 묻힙니다.

▸ WARN 포트폴리오 과잉 포장

화려한 Figma 목업, 부풀린 이력서. 실제 코드 퀄리티는 배포 후에야 드러납니다.

▸ WARN 바이브 코딩 시대

AI가 코드를 쓰는 시대. 진짜 실력은 무엇을 만드느냐, 어떤 결과를 내느냐로 결정됩니다.

▸ WARN 영업력 > 실력

말 잘하는 에이전시가 일감을 독점합니다. 조용한 실력자는 기회조차 얻지 못합니다.

✓ OK

결과물이 이력서를 대체합니다

Vibes는 포트폴리오 대신 실제 MVP 빌드 결과를 기준으로 매칭합니다.

matching-engine.diff
@@ 기존 외주 매칭 @@
- 화려한 포트폴리오로 클라이언트 현혹
- 에이전시 영업력이 프로젝트 수주 좌우
- 배포 후에야 실력 검증 (이미 늦음)
 
@@ Vibes 매칭 @@
+ 동일 과제를 블라인드로 빌드
+ 실제 MVP 결과물만으로 실력 검증
+ 클라이언트가 결과물 보고 직접 선택
+ 매칭 후에도 마일스톤 기반 검증

"개발자는 인간 AI처럼 일합니다. 프롬프트(요구사항)를 받고, 결과물을 내놓죠."

— Vibes의 매칭 철학

❯ vibes --help

4단계로 완성되는 블라인드 MVP 매칭

COMMANDS
init
요구사항 등록

클라이언트가 원하는 MVP의 핵심 기능과 기술 스택을 등록합니다. 예산과 일정도 함께 설정합니다.

match
블라인드 빌드 과제

매칭된 개발자들이 동일한 미니 과제를 블라인드로 수행합니다. 이름, 경력, 포트폴리오 — 모두 비공개.

build
결과물 평가

클라이언트가 블라인드 결과물을 비교 평가합니다. 코드 품질, 완성도, 속도 — 결과물이 전부입니다.

select
매칭 & 프로젝트 시작

선택된 개발자와 프로젝트를 시작합니다. 마일스톤 기반으로 진행하며, 결과물로 계속 검증합니다.

웨이트리스트 등록

vibes register
